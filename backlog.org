The Containerfile uses =--init none= so no systemd manages the daemon.
Currently the daemon auto-starts via =sudo= on the first nix command in each

transient =SQLite database is busy= warnings.

early in the shell profile to avoid startup races and concurrent DB contention.

*** TODO Track Emacs PGTK2/GTK4 for display reconnection support      :p2:infra:
:PROPERTIES:
:EFFORT: 0:15
:HANDOFF:
:WORKED_BY:
:END:

PGTK (GTK3) Emacs cannot survive Wayland display disconnects — the daemon
simply exits. This affects =emacs --daemon= + =emacsclient= workflows when
undocking, switching displays, or if the compositor restarts.

Emacs 31 is expected to ship with GTK4/PGTK2 which handles display reconnection.
Monitor nixpkgs for =emacs-pgtk2= or =emacs-gtk4= and switch when available.

Until then, avoid relying on long-lived daemon sessions surviving display changes.
